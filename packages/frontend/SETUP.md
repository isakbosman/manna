# Frontend Setup Instructions\n\nThis document provides setup instructions for the Manna Financial Platform frontend application.\n\n## Prerequisites\n\n- Node.js 18.0.0 or later\n- Package manager: pnpm (recommended), yarn, or npm\n- Backend API running on http://localhost:8000\n\n## Installation Steps\n\n### 1. Install Dependencies\n\nFrom the root of the monorepo:\n\n```bash\n# Using pnpm (recommended)\npnpm install\n\n# Or using yarn (if workspaces are configured)\nyarn install\n\n# Or using npm (may require additional workspace setup)\nnpm install --workspaces\n```\n\n### 2. Environment Configuration\n\nCopy the sample environment file:\n\n```bash\ncp packages/frontend/.env.sample packages/frontend/.env.local\n```\n\nConfigure the following variables in `.env.local`:\n\n```env\n# Required\nNEXT_PUBLIC_API_URL=http://localhost:8000\nNEXT_PUBLIC_APP_URL=http://localhost:3000\n\n# Optional\nNEXT_PUBLIC_APP_ENV=development\nNEXT_PUBLIC_ENABLE_ANALYTICS=false\nNEXT_PUBLIC_PLAID_ENV=sandbox\n```\n\n### 3. Start Development Server\n\n```bash\n# From the root directory\npnpm --filter @manna/frontend dev\n\n# Or from the frontend directory\ncd packages/frontend\npnpm dev\n```\n\nThe application will be available at http://localhost:3000\n\n## Package Manager Issues\n\nIf you encounter issues with package managers:\n\n### pnpm Issues\n\n1. Enable corepack (if using Node.js 16.9+):\n   ```bash\n   corepack enable\n   ```\n\n2. If corepack fails, install pnpm globally:\n   ```bash\n   npm install -g pnpm\n   ```\n\n### Workspace Protocol Issues\n\nThe project uses `workspace:*` protocol for internal dependencies. This requires:\n\n- pnpm (recommended)\n- yarn with workspaces enabled\n- npm 7+ with workspaces support\n\n### Alternative Setup (if package managers fail)\n\n1. Install dependencies manually in each package:\n   ```bash\n   # Install shared package first\n   cd packages/shared\n   npm install\n   \n   # Then install frontend (after updating package.json)\n   cd ../frontend\n   # Replace \"@manna/shared\": \"workspace:*\" with \"@manna/shared\": \"file:../shared\"\n   npm install\n   ```\n\n## Verification\n\nOnce setup is complete, verify the installation:\n\n```bash\n# Type checking\npnpm type-check\n\n# Linting\npnpm lint\n\n# Build test\npnpm build\n```\n\n## Troubleshooting\n\n### Common Issues\n\n1. **\"Cannot find module '@manna/shared'\"**\n   - Ensure the shared package is installed\n   - Check that workspace dependencies are resolved\n\n2. **\"Module not found: Can't resolve '@/...'\"**\n   - Verify tsconfig.json paths are correct\n   - Check that src directory structure matches imports\n\n3. **API connection errors**\n   - Verify backend is running on port 8000\n   - Check CORS settings in backend\n   - Confirm environment variables are set\n\n4. **TypeScript errors**\n   - Run `pnpm type-check` to see specific issues\n   - Ensure all dependencies have proper TypeScript support\n\n### Development Mode Issues\n\n1. **Hot reload not working**\n   - Try `pnpm dev:turbo` for faster development\n   - Clear Next.js cache: `rm -rf .next`\n\n2. **Style issues**\n   - Verify Tailwind CSS is properly configured\n   - Check global CSS imports in layout.tsx\n\n## Architecture Overview\n\nThe frontend is built with:\n\n- **Next.js 14+**: React framework with App Router\n- **TypeScript**: Type safety and better developer experience\n- **Tailwind CSS**: Utility-first CSS framework\n- **React Query**: Server state management\n- **React Hook Form**: Form handling with validation\n- **Zod**: Schema validation\n- **Axios**: HTTP client with interceptors\n\n## File Structure\n\n```\npackages/frontend/\n├── src/\n│   ├── app/              # Next.js App Router pages\n│   ├── components/       # Reusable React components\n│   ├── hooks/           # Custom React hooks\n│   ├── lib/             # Utilities and API clients\n│   └── styles/          # Global styles (if any)\n├── public/              # Static assets\n├── .env.sample          # Environment variables template\n├── next.config.mjs      # Next.js configuration\n├── tailwind.config.ts   # Tailwind CSS configuration\n├── tsconfig.json        # TypeScript configuration\n└── package.json         # Dependencies and scripts\n```\n\n## Next Steps\n\nAfter successful setup:\n\n1. Visit http://localhost:3000\n2. Navigate to http://localhost:3000/auth/login\n3. Use demo credentials: demo@manna.com / demo123\n4. Explore the dashboard and financial features\n\nFor detailed API integration, see the backend documentation in `packages/backend/README.md`."
